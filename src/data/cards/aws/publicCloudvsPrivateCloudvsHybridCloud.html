Public Cloud vs Private Cloud vs Hybrid Cloud

    Cloud deployment models define where your infrastructure is hosted and who can access it.
    There are three main deployment models:
        Public Cloud
        Private Cloud
        Hybrid Cloud

Public Cloud

    A Public Cloud is a cloud environment where computing resources are owned and 
    managed by a cloud provider and delivered over the internet to multiple customers.
    The infrastructure is shared among many organizations (called a multi-tenant 
    environment), but each customer's data is securely isolated.
    Examples of public cloud providers:
        AWS
        Microsoft Azure
        Google Cloud Platform (GCP)
    
    Characteristics
        Accessible over the internet
        Pay-as-you-go pricing
        Highly scalable
        No hardware maintenance
        Fast provisioning
        Managed by the cloud provider
    
    Advantages
        Low initial cost
        Easy to scale
        High availability
        Quick deployment
        Global infrastructure
    Disadvantages
        Less control over the underlying infrastructure
        Some organizations have strict compliance requirements that may limit public cloud usage

    Real-Life Analogy
        Think of a public bus.
            Many people use the same bus.
            Everyone has their own seat.
            The bus company owns and maintains the bus.
    Similarly, AWS owns the infrastructure, while multiple customers securely share it.

Private Cloud

    A Private Cloud is cloud infrastructure dedicated to a single organization.
    The servers are not shared with other companies.
    The infrastructure may be:
        Hosted in the company's own data center
        Hosted by a third-party provider but dedicated exclusively to that organization
    Characteristics
        Single organization
        Dedicated infrastructure
        Greater control
        Higher security
        Easier to meet strict compliance requirements
        Higher cost
    Example
        A bank keeps all customer data in its own private cloud because regulations 
        require complete control over infrastructure and security.
    Advantages
        Maximum control
        Enhanced security
        Better compliance support
        Greater customization
    Disadvantages
        Expensive
        Hardware maintenance is required
        Scaling is slower than in a public cloud
    Real-Life Analogy
        Imagine owning a private house.
        Only your family lives there.
    You are responsible for:
        Maintenance
        Security
        Repairs
    That's a private cloud.

Hybrid Cloud

    A Hybrid Cloud combines a Private Cloud with one or more Public Cloud environments, 
    allowing applications and data to move between them securely.
    Organizations use each environment for workloads that best fit its strengths.
    Example
    A healthcare company:
        Stores patient records in a private cloud to satisfy compliance requirements.
        Hosts its public website and appointment booking system on AWS.
    Sensitive data remains private, while customer-facing applications benefit from the 
    scalability of the public cloud.
    Advantages
        Flexibility
        Better cost optimization
        Improved disaster recovery
        Easy migration to the cloud
        Keep sensitive workloads private while scaling public-facing services
    Disadvantages
        More complex architecture
        More challenging networking and security management
        Higher operational complexity
    Real-Life Analogy
        Think of working from both home and the office.
            Sensitive meetings happen in the office.
            Regular work is done from home.
        You use both environments based on the task.
    That's a hybrid cloud.

    Memory Trick
        🚌 Public Cloud = Public bus (shared by many users)
        🏠 Private Cloud = Your own house (dedicated to you)
        🏢🏠 Hybrid Cloud = Home + Office (use both environments depending on the need)
