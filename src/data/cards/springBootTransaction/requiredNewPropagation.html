REQUIRED_NEW Propegation

    Rule : Always Create a NEW transaction.
    If another transaction exists.
    Suspend it.

Example

    @Trasactional
    publi void placeOrder{
        paymentService.payment();
        auditService.audit();
    }

    @Transactional(propegation = Propegation.REQUIRED_NEW)
    public void audit(){

    }

    BEGIN T1 -> payment() ->Suspend T1 -> BEGIN T2 ->audit()
    ->Commit T2 ->Resume T1 ->Commit T1

    Note :- There are two Transactions.

Why use REQUIRED_NEW

    Suppose Order fails.
    But audit log should always be saved.
    order ->fails ->Rollback
    Audit -> Saved Sucessfully
    Because it has its own transaction.

REQUIRED vs REQUIRES_NEW

REQUIRED	                        REQUIRES_NEW
Join existing	                    Always new transaction
One transaction	                    Multiple transactions
Parent rollback rolls back child    Independent transaction
