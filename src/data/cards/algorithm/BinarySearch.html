Binary Search finds an element in a sorted array/list by repeatedly dividing the search space into two halves.

-> Time : O(log n)
-> Space : O(1)
Pseudo Code
low = 0
high = n - 1

while (low <= high)
     mid = low + (high - low) / 2

    if arr[mid] == target
        return mid

    if arr[mid] < target
        low = mid + 1
    else
        high = mid - 1

return -1

Recursive Binary Search

Same logic, expressed recursively.
Time : O(log n)
Space : O(log n ) -> Call Stack
Pseudo Code
low = 0
high = n - 1

FUNCTION recursiveBinarySearch(low, high)
    if low > high
        return -1
    
    mid = low +(high-low)/2

    if arr[mid] == target
        return mid
    
    if arr[mid] < target
        return recursiveBinarySearch(mid+1,high)
    else
        return recursiveBinarySearch(low, mid-1)

Rotated Binary Search

A Rotated Binary Search is used to search an element in a sorted array that has been rotated.
Original: [1,2,3,4,5,6,7]
Rotated:  [4,5,6,7,1,2,3]
Instead of O(n) linear search, we can still find the element in O(log n) using binary search.
Pseudo Code
low = 0
high = n - 1
while(low <= high):
    mid = low + (high+low)/2
    if(arr[mid] == target)
        return mid
    if(left half is sorted )
        check whether target lies in left half
        yes -> high = mid -1;
        no -> low = mid+1
    elsw(right half is sorted)
        check whether target lies in right half
        yes -> low = mid+1
        no -> hig = mid-1
