Java Arrays — Interview Cheat Sheet

Core concepts · Pitfalls · Edge cases
Fundamentals
memory Array storage in memory
Arrays are objects stored on the heap, even for primitives like int[] or char[]. The reference variable lives on the stack (or as a field of another object) and points to a contiguous memory block. Size is fixed at creation and stored in the object's metadata.
core Why index starts at 0, not 1
An array index is a memory offset. Element at index i lives at address base + i × elementSize. The first element is 0 bytes from the base, so its offset = 0. Starting at 1 would force an unnecessary subtraction on every single access — wasted CPU work with no benefit.
core Default values of array elements
int / long / short / byte → 0  ·  float / double → 0.0  ·  char → '\u0000'  ·  boolean → false  ·  Object references (String[], custom types) → null
Array types
1D array
int[] arr = new int[5];
Single row of elements.
2D rectangular
int[][] arr = new int[3][4];
Array of arrays — all rows same length.
Jagged array
int[][] arr = new int[3][];
Each row can have a different length. Java 2D arrays are not true matrices.
memory 2D array memory layout
Java has no native matrix type. A 2D array is an array of references, each pointing to a separate 1D array on the heap. arr.length = number of rows; arr[i].length = number of columns in row i. This is why jagged arrays are possible.
Time complexity
Access by indexO(1)
Search — unsorted (linear)O(n)
Search — sorted (binary search)O(log n)
Insert / delete at endO(1) amortized
Insert / delete at beginning / middleO(n) — shifts required
Sort — Arrays.sort()O(n log n)
Sorting internals
primitives Dual-Pivot Quicksort
Used by Arrays.sort() for primitive arrays.
O(n log n) average. Not stable. In-place.
objects TimSort / MergeSort
Used by Arrays.sort() for object arrays.
O(n log n). Stable — preserves relative order of equal elements. Important when using Comparators.
Copying arrays
Arrays.copyOf(arr, n)
New array, can resize. Pads with defaults if larger than source.
Arrays.copyOfRange(arr, from, to)
Copies a sub-range into a new array. to is exclusive.
System.arraycopy(...)
Fastest — native method. Copies between existing arrays with precise control over positions and length.
arr.clone()
Shallow copy only. For Object[], copies references, not the objects themselves.
important Shallow copy vs deep copy
Shallow (clone(), Arrays.copyOf()) — copies array structure but elements still point to the same objects. Mutating an object via one array affects the other.
Deep — requires manually cloning each contained object so copies are fully independent.
Tricky pitfalls & interview traps
! Arrays.asList() on a primitive array
Arrays.asList(new int[]{1,2,3}) returns a List<int[]> with one element (the whole array), not List<Integer>. Autoboxing doesn't work on arrays — only on individual primitives. Fix: use Integer[] instead of int[].
! Arrays.asList() returns a fixed-size list
add() or remove() throws UnsupportedOperationException. It is a live view backed by the original array — calling set() mutates the source array too. Wrap with new ArrayList<>(Arrays.asList(...)) to get a mutable list.
! Array covariance → ArrayStoreException
Integer[] is-a Object[] compiles fine. But at runtime: Object[] o = new String[3]; o[0] = 42; throws ArrayStoreException. The JVM checks the actual underlying array type on every write, even through a wider reference.
! ArrayIndexOutOfBoundsException is unchecked
Extends RuntimeException — the compiler doesn't force you to catch it. Thrown at runtime when index < 0 or >= arr.length. Must guard manually with bounds checks or use proper iteration.
! Object[] can hold mixed types — but it's unsafe
Object[] arr can hold String, Integer, and custom objects together (all extend Object). Loses type safety — requires casting on retrieval and risks ClassCastException.
! NegativeArraySizeException
new int[-1] compiles cleanly but throws NegativeArraySizeException at runtime. A subtle edge case — especially in dynamic-size calculations where a variable could go negative.
Often-missed points
equals & toString Arrays don't override them
arr1 == arr2 checks reference equality, not content. arr1.equals(arr2) also checks reference (inherited from Object).
Use Arrays.equals(arr1, arr2) for 1D content comparison. Use Arrays.deepEquals(arr1, arr2) for nested / 2D arrays.
arr.toString() prints garbage like [I@1b6d3586 — use Arrays.toString(arr) or Arrays.deepToString().
utility Arrays.fill(arr, val)
Fills all elements with a value in O(n). Useful for resetting arrays or pre-populating with sentinel values like -1 or Integer.MAX_VALUE. Use Arrays.fill(arr, from, to, val) to fill a range.
binary search Arrays.binarySearch()
Works only on sorted arrays. Returns the index if found, or a negative value -(insertion point) - 1 if not found. Behavior is undefined on unsorted arrays — no exception is thrown.
varargs Varargs are arrays under the hood
void foo(int... nums) — nums is just an int[] at runtime. You can pass individual values or an explicit array — both are equivalent. Be careful: passing null to a varargs method assigns null to the array reference.
streams Arrays.stream()
Arrays.stream(arr) produces an IntStream / Stream<T> for functional-style ops (sum(), filter(), map()). For 2D arrays: flatten with Arrays.stream(arr).flatMapToInt(Arrays::stream).toArray().
core Size is immutable — not the contents
Once created, the length of an array cannot change. But the elements can be freely reassigned (unless the array reference is final, which only fixes the reference, not the contents).
