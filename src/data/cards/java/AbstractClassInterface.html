Abstraction

Abstraction is the process of hiding implimentation details and showing only the required functionality to the user.

Abstract Class

Any class declared with the abstract keyword is considered an abstract class. Both the class and abstract methods must use the abstract keyword. By using an abstract class, we can achieve 0% to 100% abstraction.

  1. An abstract class cannot be instantiated.
  2. Constructors are allowed in abstract classes.
  3. An abstract class does not necessarily need abstract methods.
  4. Final methods cannot be abstract because abstract methods are meant to be overridden.
  5. Objects cannot be created directly for an abstract class.
  6. Static methods can be defined in an abstract class.
Pseudo code
Abstract Class:

abstract class Test
    abstract void test();

class Test1 extends Test
    void test()
        print("Something")

Interface

An interface is similar to a class but is used to achieve 100% abstraction. By default, all methods are considered abstract (before Java 8), so there is no need to use the abstract keyword.

  1. If a class implements an interface but does not provide implementations for all interface methods, the class must be declared abstract.
Pseudo Code
Interface:

interface Test
    void test();

class Test1 implements Test
    void test()
        print("Something")
