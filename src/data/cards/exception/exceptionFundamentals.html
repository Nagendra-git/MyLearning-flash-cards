Exception Fundamentals

An Exception is an event that occurs during program execution that interrupts the normal flow of the program.
Exception handling allows the application to recover gracefully instead of crashing. 

    Why do we need Exception Handling?
    Without Exception Handling:
        Apllication ->Exception ->Program Terminates.
    With Exception Handling:
        Application ->Exception ->Catch Exception -> Continue execution.
    Benifits:
        1. Prevent application crashes.
        2. Providing meaningful error messages.
        3. Recover from failures.
        4. Keep code maintainable. 
    An exception is created, propagated, handled if a matching catch exist,
    or ultimately terminates the program if left unhandled.

Throwable

    Root class of all errors and exceptions.
    Everything that can be thrown must extend Throwable.

Error vs Exception

Error Exception
Serious JVM problem, Usually we don't handle these. Problems that Apllication can handle.
Examples: OutOfMemoryError, StackOverflowError Example: IOException, SQLException, NullPointerException

    🎯 Interview Takeaway:
        An exception is an object that represents an abnormal condition occurring during program execution. 
        It interrupts the normal flow of the application. 
        Java provides an exception handling mechanism using try, catch, finally, throw, 
        and throws to handle recoverable problems gracefully instead of terminating the application. 
        All exceptions inherit from Throwable, which also includes Error for JVM-level failures.
