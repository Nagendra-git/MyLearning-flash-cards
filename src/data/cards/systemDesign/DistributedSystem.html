Distributed System

Instead of one powerful computer handling everything, the work is divided among multiple computers.
To the user, it looks like one application, but behind the scenes, many servers are working together. 

Characteristics:
    1.Multiple computers work together.
    2.Servers communicate over a network.
    3.Appears as a single system to users.
    4.Can continue working even if one server fails (fault tolerance).
    5.Can handle more users by adding more servers (scalability).
Advantages
    ✅ High Scalability
    ✅ High Availability
    ✅ Better Performance
    ✅ Fault Tolerance
    ✅ Load Sharing
Disadvantages
    ❌ More complex to design
    ❌ Network failures can affect communication
    ❌ Data synchronization is difficult
    ❌ Debugging is harder
Memory Trick
    🖥️ One server = Centralized System
    🌐 Many servers working together = Distributed System
🎯 Interview Takeaway
    A Distributed System is a collection of multiple independent computers or servers that work together as a single system.
    These servers communicate over a network by exchanging messages to perform different tasks and achive a common goal.
    Distributed system improve scalability, availability, fault tolerance, and performance by distributing the workload across multiple machines.
    Examples: Netflix, Google search and WhatsApp
