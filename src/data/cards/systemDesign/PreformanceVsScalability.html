Performance

Performance is how fast a system responds to a user's request.

It measures:
1.Response time (Latency)
2.Throughput (Requests per second)
3.Resource usage (CPU, Memory)

Example:
Suppose a user opens Instagram.
Page loads in 1 second → Good performance.
Page loads in 10 seconds → Poor performance.

👉 Performance is about speed.

Scalability

Scalability is the ability of a system to handle increasing number of users or requests by adding more resources without significantly affecting performance. 

Resources can be:
1.More servers
2.More CPU
3.More Memory

Example:
A website works perfectly for 100 users.
    1. If it still works smoothly after adding more servers -> Good scalability.
    2. If it becomes slow or crashes -> Poor scalability.

👉 Performance is about growth.

🎯 Interview Takeaway
Performance refers to how quickly a system processes a request and returns a response. 
If a single user experiences slow response times, it's a performance issue. 
Scalability refers to a system's ability to handle increasing numbers of users or requests by adding resources. 
If the system performs well for a few users but becomes slow as traffic increases, it's a scalability issue. 
In simple terms, performance is about speed, while scalability is about handling growth efficiently.
