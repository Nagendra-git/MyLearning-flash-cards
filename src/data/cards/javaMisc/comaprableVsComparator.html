Comaprable Vs Comparator

Comparable and Comparator are both used for sorting objects in java, but they servefifferent purpouses. 

    | Feature               | Comparable                                  | Comparator                             |
    | --------------------- | ------------------------------------------- | -------------------------------------- |
    | Package               | `java.lang`                                 | `java.util`                            |
    | Purpose               | Defines the **natural ordering** of objects | Defines **custom ordering** of objects |
    | Method                | `compareTo(T o)`                            | `compare(T o1, T o2)`                  |
    | Modified Class        | The class itself implements it              | Separate class or lambda implements it |
    | Number of Sort Orders | Only one natural order                      | Multiple different sort orders         |
    | Used By               | `Collections.sort(list)`                    | `Collections.sort(list, comparator)`   |

Comparable(Natural Sorting)

    Use Comparable when the object has a default sorting order.
    
class Employee implements Comparable<Employee> {

    int id;
    String name;

    Employee(int id, String name) {
        this.id = id;
        this.name = name;
    }

    @Override
    public int compareTo(Employee other) {
        return this.id - other.id;
    }

    @Override
    public String toString() {
        return id + " " + name;
    }
}

Comparator(Custome Sorting)

    Suppose now you want to sort by name insted of ID.
    
        class NameComparator implements Comparator<Employee> {

    @Override
    public int compare(Employee e1, Employee e2) {
        return e1.name.compareTo(e2.name);
    }
}
    

    Yes. Comparable can compare any field, including name. 
    However, it defines only a single natural ordering for the class.
    If I need multiple sorting criteria such as by ID, name, salary, 
    or department, I would use Comparator because it allows multiple independent sorting strategies

    Example:
    
        public class Main {
	
	public static void main(String[] args) {
		
		List<ComparableExample> list = new ArrayList<ComparableExample>();
		list.add( new ComparableExample(1, "Nagendra"));
		list.add( new ComparableExample(3, "Janu"));
		list.add( new ComparableExample(2, "venky"));
		//Collections.sort(list); //need to implement the Comparable interface
		
		Comparator<ComparableExample> byId = Comparator.comparing(e -> e.id);
		Comparator<ComparableExample> byName = Comparator.comparing(e -> e.name);
		System.out.println(list);
		list.sort(byId);
		System.out.println(list);
		list.sort(byName);
		System.out.println(list);
		
	}

}
