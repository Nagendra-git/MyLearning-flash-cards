Java Terminal Operations

Terminal operations are the final operations in a Stream pipeline. They trigger the processing of all intermediate operations and produce a result or a side effect. After a terminal operation is executed, the stream is closed and cannot be reused.

All Common Terminal Operations

    | Terminal Operation | Purpose                                            | Returns                |
    | ------------------ | -------------------------------------------------- | ---------------------- |
    | `forEach()`        | Perform an action for each element                 | `void`                 |
    | `forEachOrdered()` | Ordered iteration (parallel streams)               | `void`                 |
    | `collect()`        | Collect elements into a collection or other result | Depends on `Collector` |
    | `toArray()`        | Convert stream to an array                         | `Object[]` or `T[]`    |
    | `reduce()`         | Reduce elements to a single value                  | `Optional<T>` or `T`   |
    | `count()`          | Count elements                                     | `long`                 |
    | `min()`            | Find minimum element                               | `Optional<T>`          |
    | `max()`            | Find maximum element                               | `Optional<T>`          |
    | `findFirst()`      | Get the first element                              | `Optional<T>`          |
    | `findAny()`        | Get any element (useful in parallel streams)       | `Optional<T>`          |
    | `anyMatch()`       | Check if any element matches                       | `boolean`              |
    | `allMatch()`       | Check if all elements match                        | `boolean`              |
    | `noneMatch()`      | Check if no elements match                         | `boolean`              |

    1.forEach():
        return void.
    
List.of(1,2,3).stream().forEach(System.out::println);
    
    2.orEachOrdered():
        Maintains encounter order, especially with parallel streams.
        Returns: void
        
numbers.parallelStream().forEachOrdered(System.out::println);
        
    3.collect():
        Most commonly used terminal operation.
        Returns: Depends on the collector (List, Set, Map, String, etc.)
        
List<String> list =names.stream().collect(Collectors.toList());
        
    4.toArray():
        Returns: Array
        
Object[] arr = names.stream().toArray();
String[] arr = names.stream().toArray(String[]::new);
        
    5.reduce():
        Combine all elements into one result.
        Returns: T (with identity) or Optional<T> (without identity)
        
int sum =List.of(10,20,30).stream().reduce(0, Integer::sum);
        
    6.count():
        Returns: long
        
long count = names.stream().count();
        
    7.min():
        Returns: Optional<T>
        
Optional<Integer> min =numbers.stream().min(Integer::compareTo);
        
    8.max():
    Returns: Optional<T>
    
Optional<Integer> max =numbers.stream().max(Integer::compareTo);
    
    9.findFirst():
        Returns: Optional<T>
        
Optional<String> first =names.stream().findFirst();
        
    10.findAny():
        Returns: Optional<T>
        May return any matching element, especially in parallel streams.
    
Optional<String> any =names.parallelStream().findAny();
    
    11.anyMatch():
        Returns: boolean
    
boolean result =numbers.stream().anyMatch(n -> n > 50);
    
    12.allMatch():
        Returns: boolean
    
boolean result =numbers.stream().allMatch(n -> n > 0);
    
    13.noneMatch():
        Returns: boolean
    
boolean result =numbers.stream().noneMatch(n -> n < 0);

Stream Lifecycle

Collection
    ↓
stream()
    ↓
Intermediate Operations
(filter → map → sorted → distinct → limit ...)
    ↓
Terminal Operation
(collect, count, reduce, findFirst, anyMatch ...)
    ↓
Result (List, Set, Map, Optional, boolean, long, etc.)

Interview Points:
    Intermediate operations are lazy; they don't execute until a terminal operation is invoked.
    Terminal operations trigger the execution of the entire stream pipeline.
    A stream can have multiple intermediate operations but only one terminal operation.
    After a terminal operation executes, the stream is consumed and cannot be reused.
