Intermediate Operations

Intermediate operations transform or filter the elements in a stream. 
    They are lazy, meaning they do not execute until a terminal operation is called. 
    Each intermediate operation returns another Stream, allowing method chaining.
    Characteristics:
        ✅ Return a new Stream
        ✅ Lazy execution
        ✅ Can be chained
        ✅ Executed only when a terminal operation is invoked

Common Intermediate Operations

    | Operation                | Purpose                                  | Returns     |
    | ------------------------ | ---------------------------------------- | ----------- |
    | `filter()`               | Filter elements                          | `Stream<T>` |
    | `map()`                  | Transform elements                       | `Stream<R>` |
    | `flatMap()`              | Flatten nested streams                   | `Stream<R>` |
    | `distinct()`             | Remove duplicates                        | `Stream<T>` |
    | `sorted()`               | Sort elements                            | `Stream<T>` |
    | `peek()`                 | Perform an action (mainly for debugging) | `Stream<T>` |
    | `limit()`                | Limit the number of elements             | `Stream<T>` |
    | `skip()`                 | Skip the first `n` elements              | `Stream<T>` |
    | `takeWhile()` *(Java 9)* | Take elements while a condition is true  | `Stream<T>` |
    | `dropWhile()` *(Java 9)* | Drop elements while a condition is true  | `Stream<T>` |

    1.filter():
        Keeps only elements that match a condition.
        
List<Integer> list = List.of(10, 15, 20, 25);
list.stream()
    .filter(n -> n % 2 == 0)
    .forEach(System.out::println);
        
    2.map():
        Transforms each element into another value.
    
List<String> names = List.of("John", "Alice");
names.stream()
     .map(String::toUpperCase)
     .forEach(System.out::println);
    
    3.flatMap():
        Flattens nested collections into a single stream.
    
List<List<String>> list = List.of(
    List.of("A", "B"),
    List.of("C", "D")
);
list.stream()
    .flatMap(List::stream)
    .forEach(System.out::println);
    
    Interview Tip:
        map() → One input produces one output.
        flatMap() → One input can produce zero, one, or many outputs and flattens them into a single stream.
    4. distinct():
        Removes duplicate elements.
    
List<Integer> list = List.of(1, 2, 2, 3, 3, 4);
list.stream()
    .distinct()
    .forEach(System.out::println);
    
    5. sorted():
        Natural sorting:
        
List<Integer> list = List.of(4, 2, 1, 3);
list.stream()
    .sorted()
    .forEach(System.out::println);
        
        Custom sorting:
        
employees.stream()
        .sorted(Comparator.comparing(Employee::getName))
         .forEach(System.out::println);
        
    6. peek():
        Used mainly for debugging.
        Without the terminal operation (count()), peek() won't execute.
        
List.of(1, 2, 3)
    .stream()
    .peek(System.out::println)
    .count();
        
    7.limit():
        Returns only the first n elements.
        
List.of(1,2,3,4,5)
    .stream()
    .limit(3)
    .forEach(System.out::println);
        
    8.skip():
        Skips the first n elements.
        
List.of(1,2,3,4,5)
    .stream()
    .skip(2)
    .forEach(System.out::println);
        

    9. takeWhile() (Java 9)
        Takes elements until the condition becomes false.
        
List.of(2,4,6,7,8)
    .stream()
    .takeWhile(n -> n % 2 == 0)
    .forEach(System.out::println);
        
    10. dropWhile() (Java 9):
        Drops elements while the condition is true, then returns the rest.
    
List.of(2,4,6,7,8)
    .stream()
    .dropWhile(n -> n % 2 == 0)
    .forEach(System.out::println);

Real-World Example

List<String> names = List.of( "John", "Alice", "David", "John", "Bob" ); List<String> result = names.stream() .filter(name -> name.length() > 3) .map(String::toUpperCase) .distinct() .sorted() .limit(3) .collect(Collectors.toList()); System.out.println(result);
