Collection Interface

    1.Root interface of Java Collection Framework.
    2.Represents a group of objects(elements).
    3.Used to store and manipulate multiple objects.
    4.Implimented by List, Set, and Queue.
        Collection = Group of Objects
    
    Hierarchy:
                Iterable
                    │
              Collection
        ┌────────┼────────┐
      List      Set      Queue
    
    Declaration:
        public interface Collection<E> extends Iterable<E>
        1.E -> Generic Type.
        2.Extends Iterable -> Supports for-each loop.
    
    Why?
        ✅ Store multiple objects
        ✅ Perform CRUD operations
        ✅ Iterate through elements
        ✅ Search elements

Collection methods

    1.add(): Add one element
        collection.add("java");
        return : boolean
    
    2. addAll(): Add all elements from another collection.
        list1.addAll(list2);
        return : boolean
    
    3.remove(): remove one element.
        list.remove("Java");
        return : boolean
    
    4.removeAll(): Removes all matching elements.
        list.removeAll(list2);
        return : boolean
    
    5.returnAll(): Keeps only common elements.( retainAll = Intersection).
        list.reatinAll(list1);
        return : boolean
    
    6.clear() : Remove everthing.
        collection.clear();
        return : void
    
    7.contains(): Checks if element exists.
        list.contains("Java");
        return : boolean
    
    8.containsAll(): Checks whether all element exists.
        list.containsAll(list2);
        return : boolean
    
    9.size(): 
        list.size() 
        return : int
    
    10.isEmpty():
        list.isEmpty()
        return : boolean
    
    11. iterator() : Used to traverse collections.

    12. toArray(): Converts Collection -> Array
        Object[] arr = list.toArray();

    13.removeIf() :Removes elements matching a condition.
        list.removeIf(n -> n%2 ==0)
    
    14.equals(): Checks whether two collections are equal.
        list1.equals(list2)
    
    15. hashCode(): return hash value of collection.
        Mostly used internally in hashing.

    16.spliterator(): 
        Used for
            Parallel Processing
            Streams
            Efficient Traversal
