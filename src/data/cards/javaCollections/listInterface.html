List Interface

    Child interface of Collection.
    Stores elements in insertion order.
    Allows duplicate elements.
    Allows index-based access.
    Remember;
        List = Ordered + Duplicates + Index
    
    Hierarchy:
            Iterable
                │
            Collection
                │
                List
             ┌──┼───────────────┐
    ArrayList  LinkedList    Vector
                                │
                                Stack

    Declaration:
        public interface List<E> extends Collection<E>

    Features:
        ✅ Ordered
        ✅ Duplicate elements allowed
        ✅ Null values allowed
        ✅ Index-based access
    
    Create a List:
        List<String> list = new ArrayList<>();
        List<String> list = new LinkedList<>();
    Always code to the interface:
        List<String> list = new ArrayList<>();
        ArrayList<String> list = new ArrayList<>();

List methods

    1.add() : Adds element at the end.
        list.add("Java");
        list.add("Spring");
    Output
        [Java, Spring]
    
    2.add(index, element): Insert at a specific position.
        list.add(1, "SQL");
    Output
        [Java, SQL, Spring]
    
    3.get(): Fetches an element using its index.
        list.get(0);
    Returns : Java(ele)

    4.set(): Replaces an existing element.
        list.set(1, "AWS");
    Before
        [Java, SQL, Spring]
    After
        [Java, AWS, Spring]

    5.remove(): By index
        list.remove(1);
    By object
        list.remove("AWS");
    6.indexOf(): First occurrence of an element.
        list.indexOf("Spring");
    Returns : 2

    7.lastIndexOf(): Returns the last occurrence.
        list.lastIndexOf("Java");
        Useful when duplicates exist.

    8.contains()
        list.contains("Java");
    Returns : boolean
    9.subList()
        list.subList(1,3);
    Example: Original
        [Java, SQL, Spring, AWS]
    Result:
        [SQL, Spring]
    End index is exclusive.
    
    10.sort()
        list.sort(String::compareTo);
    Output
        [AWS, Java, SQL, Spring]
    
    11.replaceAll()
        list.replaceAll(String::toUpperCase);
    Output
        [JAVA, SQL, SPRING]
    12.forEach(): Prints every element.
        list.forEach(System.out::println);
    
    13.ListIterator
        ListIterator<String> itr = list.listIterator();
        ✅ Move forward
        ✅ Move backward
        ✅ Add elements
        ✅ Remove elements
        ✅ Update elements
        Unlike Iterator, it is bidirectional.

Common List Implementations

| Class        | Best For                                 |
| ------------ | ---------------------------------------- |
| `ArrayList`  | Fast random access, frequent reads       |
| `LinkedList` | Frequent insertions/deletions            |
| `Vector`     | Thread-safe (legacy)                     |
| `Stack`      | LIFO operations (legacy; prefer `Deque`) |

⭐ Important List Methods

| Method           | Purpose                   |
| ---------------- | ------------------------- |
| `add()`          | Add element               |
| `add(index,e)`   | Insert at index           |
| `get()`          | Retrieve element          |
| `set()`          | Replace element           |
| `remove()`       | Delete element            |
| `contains()`     | Check existence           |
| `indexOf()`      | First occurrence          |
| `lastIndexOf()`  | Last occurrence           |
| `subList()`      | Get a portion of the list |
| `sort()`         | Sort the list             |
| `replaceAll()`   | Update all elements       |
| `listIterator()` | Bidirectional traversal   |
| `size()`         | Number of elements        |
| `isEmpty()`      | Check if empty            |
| `clear()`        | Remove all elements       |
